# 10.5 Test `ErrorBoundary` + retry

## What to build

React Testing Library tests for `ErrorBoundary` from 5.1:

1. **Catch**: child throws on render → fallback render-prop is called with `{ error, retry }`
2. **Retry**: clicking retry re-mounts the child; if the child's throwing flag has been flipped, the child renders successfully on retry

Use a controllable test component (`ThrowOnce` / `ThrowToggle`) — don't fight React's error logging in tests; just assert behavior.

Refs: `sepc.md` §15.

## Acceptance criteria

- [ ] 2 passing tests covering catch and retry
- [ ] Uses a controllable test child, not a global flag
- [ ] Retry verifiably clears the cached error state and re-mounts (assertion: post-retry, child element is in the DOM)
