# 10.4 Test `bootstrapRemotes`

## What to build

Unit tests for `host/lib/manifest.ts::bootstrapRemotes()`. Mock `fetch` and `localStorage` at the boundary. Three cases:

1. **Network success**: fetch returns manifest → `localStorage` is written under `mfe:remotes:lkg`; `init()` called with fresh remotes; `usedCache: false`
2. **Network failure + cache present**: fetch rejects/500s → `init()` called with cached remotes; `usedCache: true`
3. **Network failure + cache empty**: fetch rejects + `localStorage` empty → `init()` called with `[]`; resolves (does NOT throw)

Each case maps to a documented failure tier — these are the load-bearing tests for the failure-modes story.

Refs: `sepc.md` §15.

## Acceptance criteria

- [ ] 3 passing tests covering the three cases
- [ ] `fetch` mocked at the boundary (not Refers to internals)
- [ ] `localStorage` mocked at the boundary
- [ ] `@module-federation/runtime`'s `init` stubbed and assertions made on its call arguments
