# 11.2 Per-package workflows

## What to build

Five GitHub Actions workflow files, one per package + one for shared changes:
- `host.yml` — triggers on `packages/host/**`, `packages/shared-*/**`, `.github/workflows/host.yml`
- `remote-products.yml` — similar, scoped to remote-products
- `remote-cart.yml` — scoped to remote-cart
- `remote-orders.yml` — scoped to remote-orders
- `shared.yml` — triggers on `packages/shared-ui/**`, `packages/shared-store/**`

Each calls `_node-build.yml` with its package as the input.

Refs: `sepc.md` §19.

## Acceptance criteria

- [ ] 5 workflow files exist with the documented `paths:` filters
- [ ] Each uses `uses: ./.github/workflows/_node-build.yml` with the appropriate `package` input
- [ ] Changing only `packages/remote-cart/**` triggers only `remote-cart.yml` (verifiable via dry-run or test PR)
- [ ] Changing `packages/shared-store/**` triggers `shared.yml` AND all consumer workflows (since they're filtered on shared paths too)
