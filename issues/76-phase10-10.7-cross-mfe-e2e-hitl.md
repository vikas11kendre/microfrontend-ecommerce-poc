# 10.7 Cross-MFE E2E (the killer test)  (HITL — human verification of the full flow)

## What to build

A single Playwright test that walks the full flow end-to-end:

1. Visit `/` → redirects unauthenticated? (NO — `/` is public). Land on home with products visible.
2. Click "Login" → fill form → submit. Redirected to `/`.
3. Click a product → on detail page.
4. Click "Add to cart" → assert toast appears with product title.
5. Assert navbar cart badge shows count = 1.
6. Click cart icon → mini-cart dropdown appears with the item.
7. Navigate to `/cart` → assert item is listed.
8. Click "Place Order" → redirected to `/orders?id=ORD-...`.
9. Assert success page shows the order id.
10. Click Logout → navbar Login button reappears; mini-cart's optimistic UI cleared instantly.

**HITL** because the test exercises federation, events, redux, persistence, and middleware in one flow — a human should verify it once locally before locking it into CI.

Refs: `sepc.md` §15, §19.

## Acceptance criteria

- [ ] One Playwright test file containing the full flow above
- [ ] Test passes locally with all 4 apps running
- [ ] Assertions cover: toast visible, badge count, mini-cart contents, order id, post-logout state
- [ ] Test runs in headed mode the first time and is reviewed by a human before being marked passing
