# 10.2 Test `eventBus`

## What to build

Vitest unit tests for `shared-store`'s `emit` / `on`:
1. Emit → handler called with correct detail
2. Unsubscribe stops delivery — second `emit` does not invoke the handler
3. SSR no-op — importing in a `window`-less environment doesn't throw; `on` returns a no-op unsubscribe

Refs: `sepc.md` §15.

## Acceptance criteria

- [ ] 3 passing tests covering the three cases above
- [ ] Tests use real `CustomEvent` / `window` in jsdom (not mocked)
- [ ] SSR test runs in `node` environment (override per-file) or stubs `window` to `undefined`
