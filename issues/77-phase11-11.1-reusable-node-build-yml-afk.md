# 11.1 Reusable `_node-build.yml`

## What to build

A reusable GitHub Actions workflow at `.github/workflows/_node-build.yml` (underscore prefix is convention for `workflow_call`-only files). Accepts a `package` input. Steps:

1. Checkout
2. Setup Node (pinned version)
3. `npm ci`
4. `npm run type-check -w packages/<package>`
5. `npm run test -w packages/<package>`
6. `npm run build -w packages/<package>`
7. `node scripts/check-shared-store-version.js`

Per-package workflows (11.2) call into this single file so CI logic lives in one place.

Refs: `sepc.md` §19.

## Acceptance criteria

- [ ] `.github/workflows/_node-build.yml` exists with `on: workflow_call` and a `package` input
- [ ] All 7 steps wired in the order above
- [ ] `actionlint` (run locally) validates the file without errors
- [ ] Node version pinned (e.g., to `20.x`) consistent with sepc.md
